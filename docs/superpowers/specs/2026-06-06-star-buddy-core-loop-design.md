# 宠宠星宝轻量核心循环设计

## 背景

当前宠宠星宝已经具备基础 AI 问答入口，可以在“照顾、问问、故事、出题”之间切换，并有简单表情动画。下一步目标不是把它做成更大的聊天框，而是让 3-10 岁儿童，尤其是 6 岁左右孩子，能直觉地感觉到“星宝在陪我玩、陪我照顾动物”。

## 目标

- 星宝从工具按钮升级为会回应的陪伴角色。
- 孩子无需说明书，能通过点击、任务和奖励自然完成一次互动。
- 保留当前 AI 能力，但把 AI 回答呈现为更可点、更有反馈的卡片。
- 不引入复杂成长系统，先做轻量闭环，避免界面拥挤。

## 非目标

- 第一版不做星宝等级、亲密度、长期记忆和复杂养成。
- 第一版不新增独立“星宝小游戏房”页面。
- 第一版不让星宝替代真实动物音效。动物动作继续使用真实动物声音，星宝只使用轻快 UI 音效。

## 核心体验

孩子进入宠物照顾页后，星宝常驻在右下角或移动端底部导航上方。点击星宝身体时，星宝会做一个短动作并冒出一句简短回应。再次打开星宝时，孩子可以看到今天的 3 个小任务：

- 照顾一个：例如喝水、吃饭、休息或观察。
- 学一个：打开百科或向星宝问一个动物小秘密。
- 玩一个：完成一道星宝小题或进入游戏空间。

完成任意任务后，星宝进入庆祝表情，弹出贴纸奖励和短音效。完成全部任务后给一次更明显的“今日完成”反馈。

## 交互模块

### 1. 摸摸星宝

星宝按钮不再只有“打开菜单”一种行为。孩子点击星宝头像或身体时，先触发轻量互动：

- 随机动作：跳一下、转圈、眨眼、撒星星、歪头。
- 随机短句：例如“我在这里呢”、“一起看看星星需要什么吧”、“今天也温柔照顾小动物”。
- 如果连续点击 3 次，星宝可以做一次更明显的小庆祝，但不刷任务奖励。

交互要求：

- 动作时长控制在 600-1200ms。
- 不改变页面布局尺寸。
- 支持 `prefers-reduced-motion`，动效关闭时只换表情和短句。

### 2. 星宝入口菜单

保留当前四个入口，但文案更儿童化：

- 照顾：星宝建议下一步照顾动作。
- 问问：问一个动物小秘密。
- 故事：进入短故事卡片。
- 出题：生成一道小题。

菜单可以新增“今日任务”入口或在面板顶部直接展示任务条。第一版优先在面板顶部展示任务条，减少底部菜单按钮数量。

### 3. 今日小任务

任务条显示 3 个小目标，每个目标有状态：

- 未完成：柔和底色和图标。
- 可领取：有轻微闪动和“领贴纸”按钮。
- 已完成：显示勾和贴纸。

第一版任务可由前端根据宠物状态和当日行为生成，不必新增数据库表。可使用本地当日状态或现有用户/宠物状态派生：

- 如果孩子点击喝水，完成“照顾一个”。
- 如果打开百科或完成星宝问问，完成“学一个”。
- 如果完成星宝题目或进入游戏空间，完成“玩一个”。

如需跨设备保持任务状态，后续再扩展服务端存储。

### 4. AI 回答卡片

当前 AI 返回的 `care-plan/chat/story/quiz` 保持兼容，但前端展示增强：

- `care-plan`：显示建议卡片和大按钮，例如“去喝水”、“安静观察”。
- `chat`：显示一段短回答、2-3 个追问按钮和一个“加入今日发现”反馈。
- `story`：显示标题、2-3 段短故事、分支按钮。星宝表情随剧情变化。
- `quiz`：答题后有明显正确/错误动效。正确时触发贴纸奖励候选，错误时用温柔反馈并给解释。

AI 响应失败时，继续使用当前安全兜底内容，但也按卡片样式显示，避免像错误提示。

### 5. 贴纸奖励

完成任务或答对题时出现贴纸弹层：

- 大标题：例如“得到一枚小星星贴纸”。
- 星宝庆祝动画。
- 贴纸图案：第一版可以用 CSS/emoji/现有图形生成，不需要新素材库。
- 操作按钮：“继续照顾”、“再玩一个”。

贴纸奖励第一版可以先存在前端当日状态里，并同步到现有成就或任务计数逻辑；如果现有成就模型不适合，第一版只做视觉奖励，不改长期收藏。

### 6. 星宝短音效

星宝音效和真实动物音效分工：

- 星宝点击：轻快的叮咚、弹跳、星星声。
- 答对/完成任务：短庆祝音。
- 答错：温柔提示音。
- 动物吃饭、喝水、玩耍仍使用真实动物或真实环境声音。

音效要求：

- 音量低于动物主音效，避免吵。
- 每次点击节流，避免连续触发重叠噪音。
- 保留现有静音/有声开关。

## 状态和数据

第一版建议新增前端局部状态：

- `starMood`：星宝当前表情和动作。
- `starToast`：点击短句。
- `dailyMissions`：今日 3 个任务及状态。
- `stickerReward`：当前奖励弹层。

如需跨刷新保留，可存入 `localStorage`，key 包含用户 ID、宠物 ID、日期，避免多用户混淆。服务端暂不新增表。

## 与现有系统关系

- 继续使用现有 `StarBuddyPanel` 作为入口和 AI 面板。
- 扩展现有星宝 CSS 动画，而不是新增大型动画库。
- 继续调用 `/api/ai/star-buddy/:mode`。
- 复用现有照顾动作回调 `onCare`、百科入口 `onOpenLearn`、游戏入口 `onOpenChallenge`。
- 答题反馈可以复用此前已有的正确/错误视觉基础，再增强贴纸奖励。

## 儿童友好约束

- 单屏最多展示 3 个主要选择，避免过载。
- 关键按钮高度不低于 44px，移动端优先两列布局。
- 反馈文案短句化，不使用复杂说明。
- 错误回答不惩罚，只鼓励再试一次。
- 动效明显但不频闪。

## 错误处理

- AI 请求失败：显示星宝本地兜底卡片。
- 音效加载失败：静默跳过，不阻塞交互。
- localStorage 不可用：任务状态只保存在当前页面状态。
- 任务重复触发：同一任务当天只奖励一次。

## 验收标准

- 点击星宝头像能触发表情/动作和短句，不只是打开面板。
- 打开星宝面板能看到 3 个今日小任务。
- 完成喝水或吃饭动作后，对应任务能变成完成状态。
- 问问、故事、出题仍能正常调用 AI，并以卡片样式展示。
- 答对星宝题目后有明显庆祝反馈和贴纸弹层。
- 静音开关关闭时，星宝短音效也不播放。
- 桌面和 390px 手机宽度下，星宝面板不遮挡底部导航，内容可滚动。
- 前端类型检查、打包和后端测试通过。
